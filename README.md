<h1 align="center">Hekatē (ヘカテー)</h1>

<p align="center">
  Gemini 2.5 API の高度な機能を駆使した、自律的・記憶連動型の Discord AI アシスタント
</p>

---

## 🌟 主な特徴

### 1. 2段階ハイブリッド想起記憶 (QMD Engine)
*   **BM25 × コサイン類似度ベクトル検索**: Sudachi 形態素解析で分解したトークンによる第1段階のキーワード検索（BM25）の後、コサイン類似度によるベクトル検索（Gemini Embedding）で絞り込む高精度な想起記憶エンジン。
*   **長期記憶の自動要約（深夜バッチ）**: 毎日午前3時（JST）に過去24時間のチャットログを自動要約し、`long_term_memory.md` に蓄積。
*   **想起トークン爆発防止**: 長期記憶のサイズが肥大化してもコンテキストを圧迫しないよう、末尾から直近の一定サイズ（約2,000文字）のみをバッファ読み込みするセーフガードを搭載。

### 2. 構造化キャラクター設定 (IDENTITY / SOUL / AGENTS)
*   **パーソナリティのMarkdown分離**:
    *   `IDENTITY.md`: Bot の名前、自認、シンボル絵文字、役割の定義。
    *   `SOUL.md`: トーン、性格、話し方のルール（丁寧語ベースで少しフランクな口調など）。
    *   `AGENTS.md`: 機密情報（APIキー、環境変数）の保護やコード実行時の破壊的コマンド抑止などの安全・制約ルール。
*   **タイムスタンプ検知式ホットリロード**: Bot を再起動することなく、Markdown ファイルを編集するだけで即座にキャラクター設定や安全ルールがリアルタイムに反映。
*   **一次判定モデルでのキャラブレ防止**: fallback 不要（エスカレーションなし）の簡易返答時にも `SOUL.md` の人格が適用され、難易度を問わず一貫した口調を維持。

### 3. ハイブリッド・エスカレーション判定 (Fallback)
*   安価な一次判定モデル（`gemini-3.1-flash-lite`）で会話の難易度や応答要否を判断。
*   プログラミング、コードデバッグ、複雑な論理推論が必要な場合は、自動的にプレミアムモデル（`gemini-3.5-flash`）にタスクをエスカレーションして再生成します。

### 4. 自律 Python コード実行 (Code Execution)
*   Gemini の自律的な Python コード生成・実行機能（Code Execution Sandbox）を利用し、数値計算、データ分析、動的な画像ファイル（プロット等）の生成が可能。生成されたバイナリ画像などは自動的に Discord に添付送信されます。

### 5. 対話的・自動的スケジュールタスク登録
*   ユーザーからの自然言語の登録指示（例: 「毎週月曜日の朝9時に天気を教えて」など）から cron 式や絶対日時を抽出し、SQLite 上に定期・一回限りのタスクを自動登録・実行。
*   予定された時刻に自動で想起記憶を連動させた返答メッセージを生成、または登録されたカスタムツールを実行します。

### 6. 堅牢なリソース管理と非同期タスク制御 (Graceful Shutdown)
*   共有データベース接続（SQLite）の一元管理により、接続オープン/クローズのオーバーヘッドを削減。
*   バックグラウンドで処理される Embedding 取得タスクを安全に追跡し、シャットダウン時にすべて完了するのを待機して終了する Graceful Shutdown ロジックを実装。

## 📁 フォルダ構成

```text
[プロジェクトルート]
├── persona/             # 構造化キャラクタープロファイル定義
│   ├── IDENTITY.md      # 外面・役割・名前
│   ├── SOUL.md          # 口調・性格・回答スタイル
│   └── AGENTS.md        # 安全制限・インジェクション対策・ツール制約
├── src/                 # ソースコード
│   ├── cogs/            # Discord Cog モジュール
│   │   ├── agent_cog.py # メッセージ監視、デバウンス、LLM判定と応答、要約バッチ
│   │   └── scheduler_cog.py # スケジュール関連コマンドとポーリングループ
│   ├── agent.py         # AIAgent (Gemini API 呼び出しのカプセル化)
│   ├── context.py       # AgentContext (共有DB接続、Client、ライフサイクル管理)
│   ├── db.py            # SQLite テーブル初期化
│   ├── logger.py        # カスタムロギング初期化
│   ├── main.py          # Bot エントリーポイント
│   ├── memory_retriever.py # 短期・中期・長期記憶想起・重複排除
│   ├── qmd_engine.py    # QMD インジェスト、BM25 & ベクトルハイブリッド検索
│   ├── scheduler.py     # SQLiteベースのタスクスケジューラー
│   └── ui.py            # キャンセル View や Embed 作成などの UI コンポーネント
├── data/                # 永続化データ（DB、想起 Markdown ログ、長期記憶）
├── config.toml          # モデル名やトリガーキーワード、検索パラメータ設定
├── Dockerfile           # マルチステージ pip キャッシュ対応 Dockerfile
└── docker-compose.yml   # 開発・実行用コンパイル設定 (persona マウント対応)
```

## 🚀 セットアップと実行

### 1. 環境変数の設定
プロジェクトのルートディレクトリに `.env` ファイルを作成し、以下の環境変数を設定します。

```env
DISCORD_TOKEN=your_discord_bot_token_here
GEMINI_API_KEY=your_gemini_api_key_here
```

### 2. Docker を使用した起動
Docker Compose を使用して、コンテナをバックグラウンドでビルド・起動します。

```bash
docker-compose up -d --build
```

### 3. ホットリロードの利用
Bot の動作中に `persona/` ディレクトリ配下の Markdown ファイル（例: `SOUL.md`）の内容を直接編集して保存すると、次回の発言から即座に変更された口調や人格で応答するようになります。

---

## 🤖 スラッシュコマンド

Bot には以下のスラッシュコマンドが登録されています。

*   `/schedule_list`: 現在登録されているアクティブなスケジュールタスクの一覧を次回実行予定時刻付きで表示します。
*   `/schedule_add [cron] [instruction]`: 指定したスケジュール（cron式）で、指定した指示メッセージを実行する定期タスクを手動登録します。
*   `/schedule_delete [task_id]`: 指定された ID の定期タスクを削除します。
*   `/agent_status`: タイムゾーン、使用モデル、登録タスク数、インジェスト済みログ件数など、エージェントの内部動作ステータスを一覧表示します。

---

## 🛡️ セキュリティと安全性
*   `AGENTS.md` のルールに基づき、システムの環境変数（`DISCORD_TOKEN`, `GEMINI_API_KEY` 等）を直接的または Python コード実行経由で開示させようとするプロンプトインジェクションの試みを一次判定の段階で安全に検知し、開示を拒否します。
*   コード実行（Code Execution）における無限ループやシステム破壊的な操作を防ぐ安全なサンドボックス制限に追従しています。
